# Experience Backlog

Source: migrated from `chris-lej/school-saga-1` commit `06b6f5d0428dc5c07ec353252fc7cb52da1d803b`

This backlog tracks memorable player experiences before they become implementation issues.

## Prioritization dimensions

- **Vision fit:** Does it reward curiosity, attention, memory, or human connection?
- **Emotional value:** How strongly might the player remember it?
- **System leverage:** Does it prove or reuse an important system?
- **Cultural specificity:** Does it make the world more recognizably Brazilian?
- **Complexity:** Low, medium, high, or unknown.
- **Vertical-slice relevance:** Now, soon, later, or hold.

## Intake workflow

When a new memory is shared:

1. Capture it in `memory-vault.md`.
2. Identify one or more possible player experiences.
3. Add them here without prematurely committing to implementation.
4. Estimate complexity and system dependencies.
5. Promote selected experiences into GitHub issues when their milestone becomes active.
